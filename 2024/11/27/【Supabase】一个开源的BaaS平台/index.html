Supabase‌是一个开源的Firebase替代品，属于后端即服务（BaaS）平台。它允许开发者快速构建和部署应用程序，而无需担心底层基础设施的复杂性。

主要特点包括：

  1. ‌数据库服务‌：Supabase使用PostgreSQL作为数据库，支持SQL和RESTful API访问。这使得开发者可以方便地进行数据库操作，如增删改查（CRUD）操作‌。
  2. ‌身份验证服务‌：Supabase提供了一个完整的认证系统，支持邮箱、手机号、第三方服务等多种登录方式，方便用户管理和验证‌。
  3. ‌实时订阅功能‌：通过WebSocket，Supabase允许实现实时数据同步，无需编写额外的代码，简化了实时数据交互的复杂性‌。
  4. ‌对象存储服务‌：Supabase提供了一个对象存储服务，可以方便地上传、下载和管理文件，适用于需要存储大量数据的场景‌。
  5. ‌边缘函数‌：支持在边缘节点上运行JavaScript函数，用于处理请求或触发事件，增强了平台的灵活性和功能‌。

架构#

Supabase 是开源工具的组合，每个工具都是为企业就绪而选择的。

如果工具和社区已经存在，并且具有 MIT、Apache 2 或等效的开放许可证，我们将使用并支持该工具。 如果该工具不存在，我们自己构建并开源它。

  • Kong是云原生 API 网关。
  • GoTrue 的是基于 JWT 的 API，用于管理用户和颁发 JWT 令牌。
  • postgREST是一个 Web 服务器，可将 PostgreSQL 数据库直接转换为 RESTful API
  • Realtime是一个 Elixir 服务器，允许您使用 websockets 监听 PostgreSQL 的插入、更新和删除。实时轮询 Postgres 的内置复制功能以查找数据库更改，将更改转换为 JSON，然后通过 Websockets 将 JSON 广播到授权客户端。
  • storage提供 RESTful 接口，用于管理存储在 S3 中的文件，使用 Postgres 管理权限。
  • postgres-meta是一个 RESTful API，用于管理您的 Postgres，允许您获取表、添加角色和运行查询等。
  • PostgreSQL 数据库是一个对象关系数据库系统，经过 30 多年的积极开发，在可靠性、功能健壮性和性能方面赢得了良好的声誉。
  • Supavisor是 Postgres 的可扩展连接池程序，允许高效管理数据库连接。

为了使系统协同工作，某些服务需要在 Postgres 数据库中进行其他配置。例如，API 和 Auth 系统需要多个默认角色和 Postgres 扩展。pgjwt

您可以在架构迁移脚本存储库.这些脚本被挂载为在启动数据库容器时自动运行。/docker-entrypoint-initdb.d
